提交GitHub

之前demo提交到GitHub上，前段时间改了很多但一直没提交。今天把它提交了。
跟老哥聊天的时候其实没什么概念，游戏demo那么多文件难道要我一个个找哪些是新加的哪些是改过的哪些是删除的？所以一股脑全部提交了。本来以为和博客提交更新差不多，但是搜了一下还是很不一样的。最起码博客没有commit（这个我理解为备注，或者只是一个解释性的说明，但是好像挺重要的，没有这个会报错）。

  1. git add .（add和.之间有个空格）;

  2. git status;

  3. git commit -m “命令名”

    跟前面说的一样，不会对代码本身做说明改动，只是一个说明，效果如下（“更新”即是我这次commit的命令名）：

  4. git push origin master

其实和网上的代码不太一样。找的教程到第三步后其实还要提交什么ssh和邮箱、以及git remote add origin仓库地址之类的，最后也是push -u origin master，我嫌麻烦直接第四步，居然成功了。
