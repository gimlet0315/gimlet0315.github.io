2022.4.8
错误记录：Filename too long的问题
原因：在windows操作系统中，允许比较长的文件名称这个设置项默认是关闭(false)的。
解决：git config –global core.longpaths true

记录一下hexo常用指令：

hexo s(等价于hexo server)：启动本地服务器，用于预览主题；
hexo n(等价于hexo new)：发布新文章；
hexo d(等价于hexo deploy)：自动生成网站静态文件，并部署到设定的仓库；
hexo clean：清楚缓存文件；
hexo g(等价于hexo generate)：hexo s(等价于hexo server)：启动本地服务器，用于预览主题；
hexo n(等价于hexo new)：发布新文章；
hexo d(等价于hexo deploy)：自动生成网站静态文件，并部署到设定的仓库；
hexo n page xxx：新建一个标题为 xxx 的页面，默认链接地址为 主页地址/xxx/
常用组合：hexo clean && hexo g && hexo d;
