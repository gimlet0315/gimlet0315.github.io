Git与SVN

发表于 2022-03-22 | 分类于 Github

准备面试，似乎有关于Linux的考察，也提到了git与svn（Subversion），所以把搜集到的资料做一下整理。

关于我对git的第一印象。因为输入指令的形式太像cmd，所以一开始我把它理解为cmd和power shell类似的工具，但是今天搜了一下才知道，git和svn同属于“版本控制工具”，而cmd和shell属于“命令行解释器”，作用是在用户和内核（硬件的上层）之间起到通讯作用（也就是传达命令）。

而“版本控制工具”的作用，则不是“用户和内核”，也就是，不是用户和内核，而是用户和文件；并且它记录的并不是文件本身（如不是对某个txt文本内容进行更改），而是记录这些改动和变化，“以便将来查阅特定版本修订情况以及回溯的系统（这句不是很理解，主要是因为我目前对git的使用还没有到回溯这个功能）”，以及“将整个项目都回退到过去某个时间点的状态，你可以比较文件的变化细节，查出最后是谁修改了哪个地方，从而找出导致怪异问题出现的原因，又是谁在何时报告了某个功能缺陷等等”也存疑，这样的话岂不是要很多容量空间？但是也看了一句，说是记录“快照”，暂按不表。

版本控制工具的主要功能有两个，一个是版本管理，这一点从我搭建博客以及向GitHub push本地文件已经体会到了；另一个就是协作开发，这一点在之前做unityDemo的时候也有所了解，因为我哥要看我的文件，所以才用到的GitHub。

下面是关于git与svn的比较。

有两个链接说的比较好，贴下来记录一下：
SVN与Git比较的优缺点差异
版本管理工具——SVN和GIT

我用git是搭建博客以及push本地文件到GitHub；svn好像之前在虚拟机上用Linux的时候用过，具体记不太清了。只记得当时因为是第一次直接输入指令，所以诸多不适应，很排斥（当然现在不能说熟练吧，只能说习惯了）。

  1. git是分布式的，svn是集中式的
    我倒是明白“分布”和“集中”的意思和区别，但是对git、svn在这方面的应用没什么接触。只能说，因为使用GitHub，我大概知道git的“分布”到底意味着什么；
  2. “GIT把内容按元数据方式存储，而SVN是按文件”（这个真不明白）；
  3. Git 分支和 SVN 的分支不同
    git的分支我用到的不是很多，因为目前我每次提交文件都只用到一个分支；svn就更不明白了，教程说“分支其实是版本库中的另外一个目录”；
  4. Git 是全球唯一版本号， SVN是全局唯一版本号
    首先，版本号是啥？是每次提交文件的编号？还是git本身的版本？“Git 对于每一次提交，通过对文件的内容或目录的结构计算出一个SHA-1 哈希值，得到一个40位的十六进制字符串，Git将此字符串作为版本号”；
  5. Git 的内容完整性要优于 SVN
    这一点第四点也提到过，因为git使用SHA-1哈希算法，所以完整性很高。

以上。目前以我的能力只能理解这些了。

Linux里面没有后缀名这一说。

这张图简直是我之前的噩梦

linux系统如何执行没有后缀名的可执行文件? - 知乎用户的回答

Windows里所有文件默认都是RWX权限，即可读可写可执行。为了区分程序和文件，需要后缀名。
但Linux里文件默认是RW权限，即可读可写，只有带X权限的文件才是程序。
如果你直接输入一个命令，Windows和Linux都会在系统默认的程序文件夹下搜索相应的程序，如果找不到就给出错误提示。不过Windows的默认搜索路径里包括了当前文件夹，而Linux基于安全方面的考虑，一般不包括当前文件夹，需要用./明确指出是在当前文件夹中进行搜索，或者写出完整路径。Linux也可以设置成默认搜索路径包括当前文件夹。

搜了一下，原来后缀并不是必要的，而是为了人性化识别，文件扩展名是可以人为设定的。有扩展名无非是省略了选择打开程序，直接用默认程序打开。在Windows中，exe后缀是可执行文件，也就是程序。

Linux的目录结构：

Linux基础入门（详细版）
/bin：是Binary的缩写，这个目录存放着系统必备执行命令
/boot：这里存放的是启动Linux时使用的一些核心文件，包括一些连接文件以及镜像文件，自己的安装别放这里
/dev：Device(设备)的缩写，该目录下存放的是Linux的外部设备，在Linux中访问设备的方式和访问文件的方式是相同的。
/etc：所有的系统管理所需要的配置文件和子目录。
/home：存放普通用户的主目录，在Linux中每个用户都有一个自己的目录，一般该目录名是以用户的账号命名的。
/lib：系统开机所需要最基本的动态连接共享库，其作用类似于Windows里的DLL文件。几乎所有的应用程序都需要用到这些共享库。
/lost+found：这个目录一般情况下是空的，当系统非法关机后，这里就存放了一些文件。
/media：linux系统会自动识别一些设备，例如U盘、光驱等等，当识别后，linux会把识别的设备挂载到这个目录下。
/misc: 该目录可以用来存放杂项文件或目录，即那些用途或含义不明确的文件或目录可以存放在该目录下。
/mnt：系统提供该目录是为了让用户临时挂载别的文件系统的，我们可以将光驱挂载在/mnt/上，然后进入该目录就可以查看光驱里的内容了。
/net 存放着和网络相关的一些文件.
/opt：这是给主机额外安装软件所摆放的目录。比如你安装一个ORACLE数据库则就可以放到这个目录下。默认是空的。
/proc：这个目录是一个虚拟的目录，它是系统内存的映射，我们可以通过直接访问这个目录来获取系统信息。
/root：该目录为系统管理员，也称作超级权限者的用户主目录。
/sbin：s就是Super User的意思，这里存放的是系统管理员使用的系统管理程序。
/selinux：这个目录是Redhat/CentOS所特有的目录，Selinux是一个安全机制，类似于windows的防火墙
/srv：service缩写，该目录存放一些服务启动之后需要提取的数据。
/sys： 这是linux2.6内核的一个很大的变化。该目录下安装了2.6内核中新出现的一个文件系统 sysfs 。
/tmp：这个目录是用来存放一些临时文件的。
/usr： 这是一个非常重要的目录，用户的很多应用程序和文件都放在这个目录下，类似与windows下的program files目录。
/var：这个目录中存放着在不断扩充着的东西，我们习惯将那些经常被修改的目录放在这个目录下。包括各种日志文件。
