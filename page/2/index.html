继承（Is-A）

发表于 2022-03-21 | 分类于 C++ ， 类与类的关系

根据一个类A创建出一个新类B的过程称为继承。其中A称为父类（基类），B称为子类（派生类）。子类除了自动具有原类的成员之外，还可以根据需要增加新成员。
派生类是一个具有别的类（即父类）的属性和行为的新类，而派生类同样可以作为其他类的基类。类的派生/继承，常用来表示类属关系，即Is-A，而并非构成。进行派生时，可对派生类做出如下几种变化：

  1. 增加新的数据成员；
  2. 增加新的函数成员；
  3. 重新定义已有的成员函数；
  4. 改变现有成员的属性。

访问控制

根类：无基类派生而来，这种类的所有成员均可访问（private、public、protected）。
而不可访问（inaccessible）成员总是由基类继承而来。即要么是基类的不可访问成员， 要么是基类的私有成员。

公有继承public

基类的公有成员和保护成员，分别称为派生类的公有成员和保护成员，从而使得派生类可以访问。但派生类成员函数无法直接访问基类的私有成员。；在类外部，派生类对象可以访问继承下来的基类公有成员。

私有继承

基类的公有成员和保护成员作为派生类的私有成员，可以被派生类的成员函数直接访问。但派生类的成员函数无法直接访问基类的私有成员。在外部，派生类无法访问基类的所有成员。
私有继承使公有成员函数性质发生了变化，被派生类封装和隐藏了，需要在派生类中增加新的外部接口。

保护继承

基类的公有成员和保护成员作为派生类的保护成员，可以被派生类的成员函数直接访问。但派生类的成员函数无法直接访问基类的私有成员。在外部，派生类无法访问基类的所有成员。
但保护继承并未完全中止基类的功能。
