三伏试玩版游戏体验

发表于 2022-03-21 | 分类于 记录 ， 游戏感想

我个人对于游戏的看法大概是，分两种游戏，一种以故事为卖点，看重的是视觉效果、交互效果和故事剧本，这更接近电影；另一种是以游戏性为卖点，比如马里奥这类。更接近一般意义上的“游戏”，即街机游戏和最早的跳皮筋。现在RPG是两者之间，主打沉浸感。

三伏和烟火一样是我心里讲故事很好的国产游戏。月光蟑螂，嗯，我觉得他最厉害的地方有三个，一个是故事编的还可以（虽然我个人认为烟火剧情题材不算多新颖，但从绝大多数好评以及似乎要被翻拍成影视剧的结果看，故事讲得至少在及格线上，能够达到不少人的标准），逻辑通，人设也没什么bug，故事立意多少有些老套，但也可以算作感人；二是美术风格，也就是立绘和场景道具这些，二次元不算多写实（除了道具特写），但意外地很有生活气息，非常接地气；第三个是从这次三伏的demo看出来的，就是视觉风格。目前我觉得2D横板恐怖游戏视觉风格天花板当然是我最喜欢的harvester game的猫夫人（坠落也不差），然后就是返校最后快结局时的那段迷幻效果。这几个是我认为不仅仅是经验和技术以及和剧情的完美融合，更重要的是堪称天才的艺术性。玩猫夫人的时候的震撼是我玩任何游戏都没有的，这不是单纯生理上的惊吓，更是精神上可以让你不断回想回味的。

而三伏，我觉得从目前demo看最了不起的一点是，这是我接触到的恐怖游戏中，色调最温和明亮的，简直是堪称治愈的画风。并且demo里没有有出现jump scare的桥段。但诡异的广告形象和石膏人，还是能让人觉得不适。视觉效果我觉得很重要的一点是，这些——也就是美术、视觉效果，这些全部都是要和剧情融合在一起的。第一章最后的那段演出，我觉得还是比较震撼的——震撼不代表就要吓人。比如我之前看海边的曼彻斯特，没什么大起大落的剧情，就是很平淡地展示，但是已经足够让人感到窒息。三伏也是，你知道它要来整个大活，但没什么炫目的技巧去暗示这一点，就是通过视角的转换揭开谜底。如果我以后做独立恐怖游戏，我的学习榜样应该就是harvester game和月光蟑螂了。

总体来说，我很喜欢烟火和三伏的一点就是，它们真的可以做到“游戏电影”，有些人可能把游戏电影归作暴雨、底特律变人或者艾迪芬奇的记忆那样，可能烟火和三伏游戏性还是强了一些不能算作“电影”。但是我还是觉得，三伏可以做到。在正式版发售前，我应该会反复回味demo最后那段的演出。

继承（Is-A）

发表于 2022-03-21 | 分类于 C++ ， 类与类的关系

根据一个类A创建出一个新类B的过程称为继承。其中A称为父类（基类），B称为子类（派生类）。子类除了自动具有原类的成员之外，还可以根据需要增加新成员。
派生类是一个具有别的类（即父类）的属性和行为的新类，而派生类同样可以作为其他类的基类。类的派生/继承，常用来表示类属关系，即Is-A，而并非构成。进行派生时，可对派生类做出如下几种变化：

  1. 增加新的数据成员；
  2. 增加新的函数成员；
  3. 重新定义已有的成员函数；
  4. 改变现有成员的属性。

访问控制

根类：无基类派生而来，这种类的所有成员均可访问（private、public、protected）。
而不可访问（inaccessible）成员总是由基类继承而来。即要么是基类的不可访问成员， 要么是基类的私有成员。

公有继承public

基类的公有成员和保护成员，分别称为派生类的公有成员和保护成员，从而使得派生类可以访问。但派生类成员函数无法直接访问基类的私有成员。；在类外部，派生类对象可以访问继承下来的基类公有成员。

私有继承

基类的公有成员和保护成员作为派生类的私有成员，可以被派生类的成员函数直接访问。但派生类的成员函数无法直接访问基类的私有成员。在外部，派生类无法访问基类的所有成员。
私有继承使公有成员函数性质发生了变化，被派生类封装和隐藏了，需要在派生类中增加新的外部接口。

单继承默认为私有继承。

保护继承

基类的公有成员和保护成员作为派生类的保护成员，可以被派生类的成员函数直接访问。但派生类的成员函数无法直接访问基类的私有成员。在外部，派生类无法访问基类的所有成员。
但保护继承并未完全中止基类的功能。

多继承

二义性产生原因：

  1. 调用不同基类的相同成员

    使用作用域运算符::消除二义性

  2. 访问共同基类的成员

    多个基类有一个共同的基类，则在派生类访问此共同基类时可能出现二义性。
    依然使用作用域运算符消除二义性。
    一个基类不能从同一个类中多次继承。
    二义性检查在访问权限检查之前进行，因此成员访问权限无法解决二义性。
