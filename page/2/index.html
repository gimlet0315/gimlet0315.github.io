马里奥奥德赛感想

发表于 2022-04-08 | 分类于 随笔

想说的挺多，先说两个点
存档
收集

存档是自动存档，除了剧情点和关卡点，想说的是地表（传送点存档）。

收集
收集要素有两大类三小累：月亮和钱币（本地货币通用金币）。月亮是开启剧情/通往下一关关卡的条件，钱币可以兑换奖励解锁成就。为什么记录呢？因为它们是关卡的奖励，也可以看作关卡的标志：当你眼前有三块钱，你就知道你要开始通关了。不过奥德赛的关卡不同，它是3D，所以不像魂斗罗或者超级马里奥那样的一个个地图关卡，而是将关卡或者“谜题”融入环境中。

发表于 2022-03-22 | 分类于 数据库MySQL

关系数据库理论

主要是一些概念

  1. 函数依赖：
    学号或身份证其中的一个可以确定一个学生，但班级、性别、专业不可以。则班级、性别、专业依赖于学号/身份证。

  2. 平凡函数依赖：
    属性集Y是X的子集，例如省份就是籍贯的子集，则说Y平凡依赖于X。比如当（学号、年龄）唯一确定的时候，年龄也就唯一确定了。平凡函数依赖没有实际意义，讨论的基本都是非平凡函数依赖。

  3. 函数依赖是一个语义范畴的概念，实际上是对现实世界中事物性质之间的一种相关性断言。

  4. 函数依赖与联系类型有关：
    假如属性X与Y是1：1的关系，则存在函数依赖X↔Y；
    假设属性X与Y是1：m的关系，则只存在函数依赖Y→X；
    假设属性X与Y是n：m的关系，则X与Y之间不存在任何函数关系；

Git与SVN

发表于 2022-03-22 | 分类于 Github

准备面试，似乎有关于Linux的考察，也提到了git与svn（Subversion），所以把搜集到的资料做一下整理。

关于我对git的第一印象。因为输入指令的形式太像cmd，所以一开始我把它理解为cmd和power shell类似的工具，但是今天搜了一下才知道，git和svn同属于“版本控制工具”，而cmd和shell属于“命令行解释器”，作用是在用户和内核（硬件的上层）之间起到通讯作用（也就是传达命令）。

而“版本控制工具”的作用，则不是“用户和内核”，也就是，不是用户和内核，而是用户和文件；并且它记录的并不是文件本身（如不是对某个txt文本内容进行更改），而是记录这些改动和变化，“以便将来查阅特定版本修订情况以及回溯的系统（这句不是很理解，主要是因为我目前对git的使用还没有到回溯这个功能）”，以及“将整个项目都回退到过去某个时间点的状态，你可以比较文件的变化细节，查出最后是谁修改了哪个地方，从而找出导致怪异问题出现的原因，又是谁在何时报告了某个功能缺陷等等”也存疑，这样的话岂不是要很多容量空间？但是也看了一句，说是记录“快照”，暂按不表。

版本控制工具的主要功能有两个，一个是版本管理，这一点从我搭建博客以及向GitHub push本地文件已经体会到了；另一个就是协作开发，这一点在之前做unityDemo的时候也有所了解，因为我哥要看我的文件，所以才用到的GitHub。
