Git与SVN

发表于 2022-03-22 | 分类于 Github

准备面试，似乎有关于Linux的考察，也提到了git与svn（Subversion），所以把搜集到的资料做一下整理。

关于我对git的第一印象。因为输入指令的形式太像cmd，所以一开始我把它理解为cmd和power shell类似的工具，但是今天搜了一下才知道，git和svn同属于“版本控制工具”，而cmd和shell属于“命令行解释器”，作用是在用户和内核（硬件的上层）之间起到通讯作用（也就是传达命令）。

而“版本控制工具”的作用，则不是“用户和内核”，也就是，不是用户和内核，而是用户和文件；并且它记录的并不是文件本身（如不是对某个txt文本内容进行更改），而是记录这些改动和变化，“以便将来查阅特定版本修订情况以及回溯的系统（这句不是很理解，主要是因为我目前对git的使用还没有到回溯这个功能）”，以及“将整个项目都回退到过去某个时间点的状态，你可以比较文件的变化细节，查出最后是谁修改了哪个地方，从而找出导致怪异问题出现的原因，又是谁在何时报告了某个功能缺陷等等”也存疑，这样的话岂不是要很多容量空间？但是也看了一句，说是记录“快照”，暂按不表。

版本控制工具的主要功能有两个，一个是版本管理，这一点从我搭建博客以及向GitHub push本地文件已经体会到了；另一个就是协作开发，这一点在之前做unityDemo的时候也有所了解，因为我哥要看我的文件，所以才用到的GitHub。

三伏试玩版游戏体验

发表于 2022-03-21 | 分类于 随笔

我个人对于游戏的看法大概是，分两种游戏，一种以故事为卖点，看重的是视觉效果、交互效果和故事剧本，这更接近电影；另一种是以游戏性为卖点，比如马里奥这类。更接近一般意义上的“游戏”，即街机游戏和最早的跳皮筋。现在RPG是两者之间，主打沉浸感。

三伏和烟火一样是我心里讲故事很好的国产游戏。月光蟑螂，嗯，我觉得他最厉害的地方有三个，一个是故事编的还可以（虽然我个人认为烟火剧情题材不算多新颖，但从绝大多数好评以及似乎要被翻拍成影视剧的结果看，故事讲得至少在及格线上，能够达到不少人的标准），逻辑通，人设也没什么bug，故事立意多少有些老套，但也可以算作感人；二是美术风格，也就是立绘和场景道具这些，二次元不算多写实（除了道具特写），但意外地很有生活气息，非常接地气；第三个是从这次三伏的demo看出来的，就是视觉风格。目前我觉得2D横板恐怖游戏视觉风格天花板当然是我最喜欢的harvester game的猫夫人（坠落也不差），然后就是返校最后快结局时的那段迷幻效果。这几个是我认为不仅仅是经验和技术以及和剧情的完美融合，更重要的是堪称天才的艺术性。玩猫夫人的时候的震撼是我玩任何游戏都没有的，这不是单纯生理上的惊吓，更是精神上可以让你不断回想回味的。

github更新

发表于 2022-03-21 | 分类于 Github

之前demo提交到GitHub上，前段时间改了很多但一直没提交。今天把它提交了。
跟老哥聊天的时候其实没什么概念，游戏demo那么多文件难道要我一个个找哪些是新加的哪些是改过的哪些是删除的？所以一股脑全部提交了。本来以为和博客提交更新差不多，但是搜了一下还是很不一样的。最起码博客没有commit（这个我理解为备注，或者只是一个解释性的说明，但是好像挺重要的，没有这个会报错）。

  1. git add .（add和.之间有个空格）;

  2. git status;

  3. git commit -m “命令名”

    跟前面说的一样，不会对代码本身做说明改动，只是一个说明，效果如下（“更新”即是我这次commit的命令名）：

  4. git push origin master

其实和网上的代码不太一样。找的教程到第三步后其实还要提交什么ssh和邮箱、以及git remote add origin仓库地址之类的，最后也是push -u origin master，我嫌麻烦直接第四步，居然成功了。
